## Quickstart

### Prerequisites

- Node.js 18+
- Supabase project (URL + anon + service role keys)
- Stripe account (test mode is fine)
- Resend account + verified domain
- (Optional) PostHog project

<Steps>
<Step title="Install dependencies">
Clone the repo and install packages.

```bash
git clone https://github.com/MohammedAlhawamdeh/kwiksaas.git
cd kwiksaas
npm install
```

<Check>
Verify with `npm run lint` to ensure dependencies are healthy.
</Check>
</Step>

<Step title="Configure environment">
Copy the example env file and fill values.

```bash
cp .env.example .env.local
```

- Supabase: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`, `SUPABASE_SECRET_KEY`
- Stripe: `STRIPE_SECRET_KEY`, `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`, `STRIPE_WEBHOOK_SECRET`, price IDs (`PRO_MONTHLY`, `PRO_YEARLY`, `LIFETIME`, `STANDARD_LIFETIME`, `ULTIMATE_LIFETIME`, `PRO_TRIAL`)
- Resend: `RESEND_API_KEY`, `RESEND_FROM_EMAIL`, `RESEND_LOGO_URL`
- Site: `NEXT_PUBLIC_SITE_URL` (e.g., `http://localhost:3000`)
- Optional: `NEXT_PUBLIC_POSTHOG_KEY`, `NEXT_PUBLIC_POSTHOG_HOST`

<Warning>
Do not commit `.env.local` or any secrets to version control.
</Warning>
</Step>

<Step title="Apply database schema">
Push migrations to Supabase.

```bash
npx supabase db push
```

Alternatively, run the SQL in `supabase/migrations/*.sql` via the Supabase SQL Editor.
</Step>

<Step title="Configure Stripe">
- Create products/prices and map price IDs into `.env.local`.
- Start a local webhook listener:

```bash
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

Set `STRIPE_WEBHOOK_SECRET` from the listener output.
</Step>

<Step title="Configure Resend">
Verify your sending domain and set `RESEND_FROM_EMAIL` and `RESEND_LOGO_URL`.
</Step>

<Step title="Run the app">

```bash
npm run dev
# open http://localhost:3000
```

<Check>
You should see the marketing homepage; auth flows should redirect correctly.
</Check>
</Step>
</Steps>
